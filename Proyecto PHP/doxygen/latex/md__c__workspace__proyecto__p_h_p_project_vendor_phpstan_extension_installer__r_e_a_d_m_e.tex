Starting from \mbox{\hyperlink{namespace_composer}{Composer}} 2.\+2.\+0 you\textquotesingle{}ll get the following question\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{phpstan/extension-\/installer contains a Composer plugin which is currently not in your allow-\/plugins config. See https://getcomposer.org/allow-\/plugins}
\DoxyCodeLine{Do you trust "{}phpstan/extension-\/installer"{} to execute code and wish to enable it now? (writes "{}allow-\/plugins"{} to composer.json) [y,n,d,?]}

\end{DoxyCode}


Answer with {\ttfamily y} to allow the plugin.\hypertarget{md__c__workspace__proyecto__p_h_p_project_vendor_phpstan_extension_installer__r_e_a_d_m_e_autotoc_md429}{}\doxysubsection{Instructions for extension developers}\label{md__c__workspace__proyecto__p_h_p_project_vendor_phpstan_extension_installer__r_e_a_d_m_e_autotoc_md429}
It\textquotesingle{}s best (but optional) to set the extension\textquotesingle{}s composer package \href{https://getcomposer.org/doc/04-schema.md\#type}{\texttt{ type}} to {\ttfamily phpstan-\/extension} for this plugin to be able to recognize it and to be \href{https://packagist.org/explore/?type=phpstan-extension}{\texttt{ discoverable on Packagist}}.

Add {\ttfamily phpstan} key in the extension {\ttfamily composer.\+json}\textquotesingle{}s {\ttfamily extra} section\+:
