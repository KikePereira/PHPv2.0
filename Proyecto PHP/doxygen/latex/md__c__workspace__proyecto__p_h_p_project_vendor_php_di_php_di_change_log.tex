Improvements\+:


\begin{DoxyItemize}
\item \href{https://github.com/PHP-DI/PHP-DI/pull/494}{\texttt{ \#494}} The container can now be compiled for optimum performances in production
\item \href{https://github.com/PHP-DI/PHP-DI/issues/294}{\texttt{ \#294}}, \href{https://github.com/PHP-DI/PHP-DI/issues/349}{\texttt{ \#349}}, \href{https://github.com/PHP-DI/PHP-DI/pull/449}{\texttt{ \#449}}\+: {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}\mbox{\hyperlink{_array_declaration_unit_test_82_8inc_a34ed4cda060ffc5ababa5e187c10a4aa}{object()}}} has been replaced by more specific and less ambiguous helpers\+:
\begin{DoxyItemize}
\item {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}create()} creates an object, overrides autowiring and previous definitions
\item {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}autowire()} autowires an object and allows to override specific constructor and method parameters
\end{DoxyItemize}
\item The container can now be built without parameters\+: {\ttfamily new Container()}
\item Definitions can be nested\+:
\begin{DoxyItemize}
\item \href{https://github.com/PHP-DI/PHP-DI/issues/490}{\texttt{ \#490}} Definitions can be nested in arrays (by \href{https://github.com/yuloh}{\texttt{ @yuloh}})
\item \href{https://github.com/PHP-DI/PHP-DI/issues/501}{\texttt{ \#501}} \& \href{https://github.com/PHP-DI/PHP-DI/issues/540}{\texttt{ \#540}} Autowire definitions can be nested in other definitions
\item \href{https://github.com/PHP-DI/PHP-DI/issues/487}{\texttt{ \#487}} \& \href{https://github.com/PHP-DI/PHP-DI/issues/540}{\texttt{ \#540}} Closures are now handled as factories when they are nested in other definitions
\end{DoxyItemize}
\item \href{https://github.com/PHP-DI/PHP-DI/issues/487}{\texttt{ \#487}} Closures in the config are now always interpreted as factories, even when nested inside other definitions
\item \href{https://github.com/PHP-DI/PHP-DI/issues/242}{\texttt{ \#242}} Error in case a definition is not indexed by a string
\item \href{https://github.com/PHP-DI/PHP-DI/pull/505}{\texttt{ \#505}} Debug container entries
\item \href{https://github.com/PHP-DI/PHP-DI/pull/564}{\texttt{ \#564}} Caching was made almost entirely obsolete by the container compilation, however there is still a caching system entirely rebuilt over APCu for covering the last cases that compilation could not address (see \href{https://php-di.org/doc/performances.html}{\texttt{ php-\/di.\+org/doc/performances.html}})
\end{DoxyItemize}

Fixes\+:


\begin{DoxyItemize}
\item \href{https://github.com/PHP-DI/PHP-DI/issues/499}{\texttt{ \#499}} \& \href{https://github.com/PHP-DI/PHP-DI/issues/488}{\texttt{ \#488}} Standardize resolution of nested definitions everywhere. In PHP-\/\+DI 5, definitions could be nested in some places (e.\+g. use a get() in an object definition, etc.). However it did not behave everywhere the same, for example it didn\textquotesingle{}t work for sub-\/definitions in arrays. Now in PHP-\/\+DI 6 all nested definitions will all be recognized and resolved correctly everywhere. Since \#494 (compiled container) performance will not be affected so we can implement a more robust behavior.
\item \href{https://github.com/PHP-DI/PHP-DI/issues/343}{\texttt{ \#343}} Autowiring and Annotations do not work for {\ttfamily \mbox{\hyperlink{_array_declaration_unit_test_82_8inc_a34ed4cda060ffc5ababa5e187c10a4aa}{object()}}} inside arrays\+: it now works with the new {\ttfamily create()} and {\ttfamily autowire()} helpers
\end{DoxyItemize}

BC breaks\+:


\begin{DoxyItemize}
\item PHP 7 or greater is required and HHVM is no longer supported
\item {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}\mbox{\hyperlink{_array_declaration_unit_test_82_8inc_a34ed4cda060ffc5ababa5e187c10a4aa}{object()}}} has been removed, use {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}create()} or {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}autowire()} instead
\item \href{https://github.com/PHP-DI/PHP-DI/issues/409}{\texttt{ \#409}}\+: Scopes are removed, read more in the scopes documentation.
\item Caching was replaced by compiling the container\+: {\ttfamily Container\+Builder\+::set\+Definition\+Cache()} was removed, use {\ttfamily Container\+Builder\+::enable\+Compilation()} instead.
\item \href{https://github.com/PHP-DI/PHP-DI/issues/463}{\texttt{ \#463}} \& \href{https://github.com/PHP-DI/PHP-DI/issues/485}{\texttt{ \#485}}\+: Container-\/interop support was removed, PSR-\/11 is used instead (by \href{https://github.com/juliangut}{\texttt{ @juliangut}})
\item The deprecated {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}link()} helper was removed, used {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}get()} instead
\item \href{https://github.com/PHP-DI/PHP-DI/pull/484}{\texttt{ \#484}} The deprecated {\ttfamily \textbackslash{}\mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Debug} class has been removed. Definitions can be cast to string directly
\item The exception {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Definition\textbackslash{}Exception\textbackslash{}Definition\+Exception} was renamed to {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Definition\textbackslash{}Exception\textbackslash{}Invalid\+Definition}
\item The exception {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Definition\textbackslash{}Exception\textbackslash{}Annotation\+Exception} was renamed to {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Definition\textbackslash{}Exception\textbackslash{}Invalid\+Annotation}
\item \href{https://github.com/PHP-DI/PHP-DI/issues/516}{\texttt{ \#516}} {\ttfamily \mbox{\hyperlink{namespace_d_i}{DI}}\textbackslash{}Invoker\+Interface} was removed in favor of {\ttfamily \mbox{\hyperlink{namespace_invoker}{Invoker}}\textbackslash{}Invoker\+Interface}.
\end{DoxyItemize}
