Parses a route string into multiple route data arrays.

The expected output is defined using an example\+:

For the route string \char`\"{}/fixed\+Route\+Part/\{var\+Name\}\mbox{[}/more\+Fixed/\{var\+Name2\+:\textbackslash{}d+\}\mbox{]}\char`\"{}, if \{var\+Name\} is interpreted as a placeholder and \mbox{[}...\mbox{]} is interpreted as an optional route part, the expected result is\+:

\mbox{[} // first route\+: without optional part \mbox{[} \char`\"{}/fixed\+Route\+Part/\char`\"{}, \mbox{[}\char`\"{}var\+Name\char`\"{}, \char`\"{}\mbox{[}$^\wedge$/\mbox{]}+\char`\"{}\mbox{]}, \mbox{]}, // second route\+: with optional part \mbox{[} \char`\"{}/fixed\+Route\+Part/\char`\"{}, \mbox{[}\char`\"{}var\+Name\char`\"{}, \char`\"{}\mbox{[}$^\wedge$/\mbox{]}+\char`\"{}\mbox{]}, \char`\"{}/more\+Fixed/\char`\"{}, \mbox{[}\char`\"{}var\+Name2\char`\"{}, \mbox{[}0-\/9\mbox{]}+"{}\mbox{]}, \mbox{]}, \mbox{]}

Here one route string was converted into two route data arrays.
